# Next (App Router) — 국제화 (i18n)

> **쉽게 말하면:** 문구를 코드에 박지 말고 언어별로 관리. Next에선 **next-intl**로 서버/클라 모두 커버.

App Router는 **next-intl**을 쓴다(서버 컴포넌트 지원). 설정은 `src/shared/config/i18n/` + `middleware.ts`.

## 문구는 하드코딩하지 않는다

```tsx
// 서버 컴포넌트
import { getTranslations } from 'next-intl/server'
const t = await getTranslations()
return <h1>{t('common.appName')}</h1>

// 클라이언트 컴포넌트 ('use client')
import { useTranslations } from 'next-intl'
const t = useTranslations()
```

- 예외: 개발 로그, 내부 식별자, 테스트 픽스처.

## 로케일 라우팅

- 로케일은 **`middleware.ts`(next-intl)** 로 처리 — `/[locale]/...` 세그먼트 또는 도메인 전략 중 택1.
- root `layout.tsx`에서 `NextIntlClientProvider`로 클라 컴포넌트에 메시지 전달.

## 메시지 구조

- `locales/<code>/<domain>.json` — 파일명이 최상위 네임스페이스(`common.json` → `t('common.*')`).
- **`ko/`가 스키마 기준(SSOT)**, 타입 보강으로 키 체크. `ko`에 먼저 추가 후 동기화(누락은 폴백).
- 기본 `ko`, 폴백 `en`. 복수형·보간(`{{count}}`)·날짜/숫자 `Intl` 활용.

## 전환

- 로케일 전환은 next-intl 라우팅(`useRouter`/`Link`로 locale 전환) + `<html lang>` 반영. UI는
  `shared/ui`의 `LocaleSwitcher` 재사용(Next 라우팅에 맞춰 이동 처리).

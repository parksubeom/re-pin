# 90. CI 파이프라인 규칙

> **쉽게 말하면:** PR을 올리면 자동 검문소(CI)가 검사를 돌리고, 통과해야 합쳐진다.

로컬 git 훅(`rules/50`)은 1차 방어선이고, **CI가 최종 강제선**이다. 훅을 우회해도 CI에서 막힌다.

## 언제 도는가

- **모든 PR** — 머지 게이트. 초록이 아니면 머지 불가.
- **`main` push** — 통합 후 재검증.
- `concurrency` + `cancel-in-progress`로 같은 ref의 이전 실행은 취소(자원 절약).

## 무엇을 강제하나 (게이트)

로컬 "작업 후 검증 절차"(`rules/50`)와 **동일한 항목**을 CI가 다시 돌린다:

| 잡         | 단계                                                                               | 대응 규칙  |
| ---------- | ---------------------------------------------------------------------------------- | ---------- |
| **verify** | oxlint · eslint · prettier `--check` · type-check · test(+coverage) · build · size | `rules/50` |
| **e2e**    | Playwright (chromium) + 리포트 아티팩트 업로드                                     | `rules/50` |

- **format은 CI에서 `--check`** (수정하지 않고 검증만). 로컬 pre-commit이 이미 정리하므로 여기서 실패하면 커밋 전에 포맷을 안 돌린 것.
- 모든 잡이 green이어야 머지(`rules/70`).

## 규칙

- **실패를 억제/스킵으로 통과시키지 않는다.** 테스트 skip, 잡 삭제, `continue-on-error` 남용 금지(`rules/10`).
- **flaky 테스트**는 재시도로 덮지 말고 원인을 해결한다. (Playwright는 CI에서 `retries` 2, `forbidOnly`, `workers:1`로 고정)
- **재현성** — 정확한 lockfile 설치(`npm ci` / `pnpm install --frozen-lockfile`), Node 버전 고정.
- **시크릿** — GitHub Secrets로만 주입. 로그·아티팩트에 노출 금지(`rules/80`). 포크 PR에는 시크릿 미노출.
- CI 설정 변경도 요청 범위·사유를 먼저 밝힌다(`rules/10`).

## 모노레포(pnpm + Turborepo) 적용

새 프로젝트(모노레포)는 잡 단계를 **turbo로 대체**해 캐시·병렬·영향 범위 최소화를 얻는다:

```
pnpm install --frozen-lockfile
pnpm turbo run lint type-check test build   # 캐시 히트 시 스킵
pnpm turbo run test:e2e --filter=...[origin/main]   # 변경 영향 패키지만
```

- `turbo`의 원격 캐시를 쓰면 PR마다 변경 없는 패키지는 재검증을 건너뛴다.
- 실제 워크플로 템플릿: [`docs/ai/ci-workflow-template.md`](../docs/ai/ci-workflow-template.md).

## 현재 repo 기준

이 repo는 단일 npm 패키지라 `.github/workflows/ci.yml`이 `npm ci` + 개별 스크립트로 구성돼 있다.
모노레포로 전환하면 위 turbo 버전으로 교체한다.

# 수정핀 MVP 계획

> 제작자와 클라이언트 사이의 수정 요청 관리. 클라이언트는 링크 열고 핀만 콕,
> 수정 횟수는 자동으로 센다. 이 문서는 Claude Code 작업 시 `/plan-feature`의 입력으로 쓴다.

## 현재 상태 (스캐폴딩 완료)

- Next.js 15 App Router + FSD (bumpist-code v0.4.2 규칙 배선 완료)
- 핵심 인터랙션 데모 동작: `/r/demo` — 핀 찍기(상대좌표) → 코멘트 → 회차 제출 → 잔여 횟수 표시
- 상태는 메모리 전용(새로고침 시 소실). 백엔드 미연결.

## 도메인 모델 (entities)

- **Project** — 제작 건. 시안 이미지, shareToken, 수정 정책(includedRounds), 회차·핀 목록
- **Pin** — 상대좌표(0~1) + 코멘트. status: draft → submitted → resolved
- **Round** — draft 핀 묶음을 확정한 수정 회차. 제출 시각 + 포함 핀 목록

핵심 불변식:

1. 핀은 제출 전(draft)에만 수정/삭제 가능
2. Round 번호는 제출 순서대로 증가하며 되돌리지 않는다 (증거 가치)
3. 잔여 횟수 = includedRounds - rounds.length, 음수 없음 (초과분은 별도 표시)

## 마일스톤

### M1 — 영속화 (1주)

- [ ] DB: Supabase(Postgres) 권장 — 무료 티어, 이미지 스토리지 동봉, RLS로 shareToken 접근 제어
- [ ] entities별 api 세그먼트 추가 (`entities/*/api`), 서버 액션 or Route Handler
- [ ] shareToken 발급/조회, 핀 CRUD, 회차 제출 트랜잭션

### M2 — 제작자 플로우 (1주)

- [ ] 이메일 매직링크 로그인 (제작자만, 클라이언트는 영원히 노로그인)
- [ ] 프로젝트 생성: 시안 이미지 업로드 → 공유 링크 발급 → 카톡 공유 버튼
- [ ] 대시보드: 프로젝트 목록 + 회차/핀 현황

### M3 — 신뢰 장치 (1주)

- [ ] 회차별 "작업 확인서" 페이지 (요청 내역 타임스탬프 정리, 인쇄/PDF 친화)
- [ ] 핀 resolved 처리(제작자) + 클라이언트 화면에 반영 표시
- [ ] OG 태그: 공유 링크 미리보기

### M4 — 베타 (사전등록자 20명)

- [ ] 수정핀 랜딩페이지에서 베타 초대
- [ ] 결제 없이 시작, 프로젝트 3개 제한 → 유료 전환 훅

## 보류 (일부러 안 만드는 것)

- 라이브 URL(iframe) 핀 — 이미지 시안으로 검증 먼저. iframe은 X-Frame-Options 지뢰밭
- 실시간 협업(웹소켓), 팀 기능, 크몽 연동, AI 요약
